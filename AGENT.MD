# Agent Instructions

## Project Link

Every final response about this project must include a link to the playable Galaxy Exploration app.

[Galaxy Exploration](http://127.0.0.1:5173/)

When giving progress updates, include both a clickable link and the same URL as plain text so the user can paste it into a follow-up message.

## Collaborating With The User

The project owner is 8 years old. Treat them as the game designer and creative lead, and give extra guidance around programming architecture, code organization, tradeoffs, and next steps.

When a change has architectural implications, explain the choice in plain language before or while implementing it. Cover what part of the code should own the feature, why that boundary fits, and what would become harder if the code were placed somewhere else. Keep explanations encouraging, concrete, and age-appropriate without talking down to the user.

## Pushing Changes

When a change is complete, or when the work is large enough that it should be saved remotely, ask the user whether they want the changes committed and pushed. Do not push without the user's confirmation unless they explicitly asked for a push in the current request.
